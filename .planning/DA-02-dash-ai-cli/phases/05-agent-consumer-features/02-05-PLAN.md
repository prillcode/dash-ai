# PLAN 02-05 — Agent Consumer Features

## Objective
Polish the CLI for first-class AI agent consumption — structured JSON output, file path returns, `wait` command, `--stdout` piping, and proper exit codes for scripting. This is the agent-facing payoff.

## Tasks

### 1. `dash-ai tasks wait <id>`

Blocking command that waits for a task to reach a terminal state:

```typescript
tasks.command("wait <id>")
  .description("Wait for task to reach a terminal state")
  .option("--timeout <seconds>", "max wait time in seconds (default: 1200 = 20 min)")
  .option("--status <status>", "wait for specific status instead of any terminal state")
  .action(async (id, opts) => {
    const ctx = getContext(opts)
    const { client } = await resolveClient(ctx)

    const targetStatuses = opts.status
      ? [opts.status]
      : ["PLANNED", "AWAITING_REVIEW", "APPROVED", "REJECTED", "COMPLETE", "FAILED"]

    const task = await pollTaskStatus(id, client, {
      targetStatus: targetStatuses,
      timeoutMs: (parseInt(opts.timeout) || 1200) * 1000,
    })

    if (ctx.json) {
      console.log(formatJson({
        taskId: task.id,
        status: task.status,
        planPath: task.planPath,
        diffPath: task.diffPath,
        errorMessage: task.errorMessage,
        completedAt: task.completedAt,
      }))
    } else {
      console.log(`Task ${task.id} is now ${task.status}`)
    }

    // Exit code: 6 if task failed
    process.exit(task.status === "FAILED" ? 6 : 0)
  })
```

Use case: an agent creates a task, triggers planning, then waits:
```bash
dash-ai tasks plan abc-123
dash-ai tasks wait abc-123 --status PLANNED
dash-ai tasks plan-docs abc-123 --file BRIEF --stdout
```

### 2. `dash-ai tasks plan-docs <id> --stdout`

Concatenates all plan docs and pipes to stdout for agent consumption:

```typescript
if (opts.stdout) {
  // Print each doc with a header separator
  const files = ["BRIEF.md", "ROADMAP.md", ...phasePlanFiles]
  for (const file of files) {
    const content = await client.get(`/api/tasks/${id}/plan-doc?file=${file}`)
    process.stdout.write(`=== ${file} ===\n`)
    process.stdout.write(content + "\n\n")
  }
  return
}
```

An agent pipes this into its context:
```bash
dash-ai tasks plan-docs abc-123 --stdout | pi -p "Review this plan and suggest improvements"
```

### 3. `dash-ai tasks diff <id> --stdout`

Pipes raw unified diff to stdout:

```typescript
if (opts.stdout) {
  const diff = await client.get(`/api/tasks/${id}/diff`)
  process.stdout.write(diff)
  return
}
```

Agent can apply it:
```bash
dash-ai tasks diff abc-123 --stdout | git apply --stat  # preview
dash-ai tasks diff abc-123 --stdout | git apply          # apply
```

### 4. Consistent JSON schema

All `--json` output follows a consistent envelope:

```typescript
// Success response
{
  "success": true,
  "data": { ... }  // command-specific payload
}

// Error response
{
  "success": false,
  "error": {
    "message": "Task not found: abc-123",
    "code": 3
  }
}
```

Command-specific JSON payloads:

### 5. `dash-ai tasks create --auto-plan`

Convenience flag that creates a task and immediately triggers planning:

```typescript
tasks.command("create")
  // ... existing options
  .option("--auto-plan", "immediately start planning after creation")
  .action(async (opts) => {
    // Create the task
    const task = await client.post("/api/tasks", { ... })

    if (opts.autoPlan) {
      // Trigger planning
      await client.post(`/api/tasks/${task.id}/start-planning`)

      if (opts.json) {
        console.log(formatJson({
          success: true,
          data: { taskId: task.id, status: "IN_PLANNING" }
        }))
      } else {
        console.log(`Task created and planning started: ${task.id}`)
        console.log(`  Watch: dash-ai tasks watch ${task.id}`)
      }
    } else {
      // ... normal create output
    }
  })
```

### 6. Reviewer persona output — file paths

When an agent calls `review --json`, the response includes file paths the agent can read directly:

```json
{
  "data": {
    "taskId": "abc-123",
    "review": { ... },
    "artifacts": {
      "diffPath": "~/.dash-ai/diffs/abc-123/changes.diff",
      "planDocsPath": "/home/user/projects/my-app/.planning/da-abc-rate-limiting/",
      "sessionLogPath": "~/.dash-ai/sessions/abc-123/session.log"
    }
  }
}
```

The calling agent can then decide to read any of these files for additional context.

### 7. Exit code enforcement

Ensure every command path sets the correct exit code:

```typescript
function exit(code: number): never {
  process.exit(code)
}

// Usage:
if (task.status === "FAILED") exit(6)
if (!found) exit(3)
if (authError) exit(4)
// ... implicit exit(0) on success
```
