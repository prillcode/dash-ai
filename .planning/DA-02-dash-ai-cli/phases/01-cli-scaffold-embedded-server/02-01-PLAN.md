# PLAN 02-01 — CLI Scaffold + Embedded Server Bootstrap

## Objective
Create the `packages/cli/` package with a working CLI framework, embedded server bootstrap, thin API client, and output formatting. After this plan, `dash-ai --help` works and the package builds cleanly.

## Tasks

### 1. Create `packages/cli/` package

```bash
mkdir -p packages/cli/src/{commands,embedded,api,output}
```

Key decisions:
- **commander** for CLI framework — well-typed, `git`-style subcommands, works in ESM
- **chalk** for terminal colors
- **cli-table3** for formatted table output

- Depends on `@dash-ai/server` as a workspace dependency — reuses all services, schema, runners

export interface CliContext {
  json: boolean
  quiet: boolean
  color: boolean
  url?: string       // DASH_AI_URL override
  token?: string     // DASH_AI_TOKEN override

  return {
    json: opts.json ?? false,

    color: opts.color ?? true,
    url: opts.url ?? process.env.DASH_AI_URL,

### 4. Embedded server bootstrap (`src/embedded/server.ts`)

This is the core of standalone mode. It spins up the Hono server + SQLite in the current process:

```typescript
import { Hono } from "hono"
import { serve } from "@hono/node-server"
import { registerRoutes } from "@dash-ai/server/routes"  // or equivalent
import { initDatabase } from "@dash-ai/server/db/client"
import { startQueueWorker } from "@dash-ai/server/services/queueWorker"

let serverInstance: ReturnType<typeof serve> | null = null
let queueWorkerStarted = false

export async function ensureEmbeddedServer(): Promise<{ url: string; token: string }> {
  if (serverInstance) {
    return { url: `http://localhost:${serverInstance.port}`, token: getApiToken() }
  }

  // Initialize database
  initDatabase()

  // Create Hono app with all routes (reuse server package)
  const app = createApp()  // shared app factory exported from server

  // Find available port
  const port = 0  // OS assigns random available port

  serverInstance = serve({ fetch: app.fetch, port })

  // Start queue worker (needed for coding tasks)
  if (!queueWorkerStarted) {
    startQueueWorker()
    queueWorkerStarted = true
  }

  return { url: `http://localhost:${serverInstance.port}`, token: getApiToken() }
}

export async function teardownEmbeddedServer(): Promise<void> {
  if (serverInstance) {
    serverInstance.close()
    serverInstance = null
  }
}
```

**Key design:** The server package needs a refactor to export a shared `createApp()` factory (or the CLI imports the route registrations directly). The server's `index.ts` currently does setup inline — extract the app creation into a reusable function.

### 8. Server resolution logic (`src/api/resolver.ts`)

Determines whether to use thin client or embedded mode:

```typescript
export async function resolveClient(ctx: CliContext): Promise<{ client: DashAiClient; embedded: boolean }> {
  const url = ctx.url || process.env.DASH_AI_URL
  const token = ctx.token || process.env.DASH_AI_TOKEN

  if (url && token) {
    // Thin client mode — remote server configured
    return { client: new DashAiClient(url, token), embedded: false }
  }

  // Embedded mode — spin up in-process server
  const { url: embeddedUrl, token: embeddedToken } = await ensureEmbeddedServer()
  return { client: new DashAiClient(embeddedUrl, embeddedToken), embedded: true }
}
```
