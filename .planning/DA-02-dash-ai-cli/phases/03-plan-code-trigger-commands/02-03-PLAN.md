# PLAN 02-03 — Plan & Code Trigger Commands

## Objective
Implement commands that trigger agent execution — planning sessions, coding queue, reviewer persona, and diff/review/approve workflows. This is where the CLI becomes "magic" — actual Pi SDK sessions fire from terminal commands.

## Tasks

### 1. `dash-ai tasks plan <id>`

Triggers a planning session via embedded Pi SDK:

```typescript
tasks.command("plan <id>")
  .description("Start planning session for a task")
  .option("--feedback <text>", "feedback for plan iteration")
  .action(async (id, opts) => {
    const ctx = getContext(opts)
    const { client } = await resolveClient(ctx)

    // Check current status
    const task = await client.get(`/api/tasks/${id}`)
    if (task.status !== "DRAFT" && task.status !== "PLANNED") {
      error(`Cannot plan task in status ${task.status}. Must be DRAFT or PLANNED.`, { exitCode: 1 })
    }

    // Trigger planning via API
    const endpoint = opts.feedback
      ? `/api/tasks/${id}/iterate-plan`
      : `/api/tasks/${id}/start-planning`

    const body = opts.feedback ? { feedback: opts.feedback } : {}
    await client.post(endpoint, body)

    // Poll for completion
    const spinner = ora("Planning in progress...").start()
    const finalTask = await pollTaskStatus(id, client, {
      targetStatus: ["PLANNED", "FAILED"],
      spinner,
      onEvent: ctx.json ? undefined : (event) => {
        // Show minimal progress (tool calls)
        if (event.type === "TOOL_START") {
          spinner.text = `Planning... (${event.toolName})`
        }
      },
    })

    spinner.stop()

    if (finalTask.status === "FAILED") {
      error(`Planning failed: ${finalTask.errorMessage}`, { exitCode: 6 })
    }

    if (ctx.json) {
      console.log(formatJson({ taskId: id, status: "PLANNED", planPath: finalTask.planPath }))
    } else {
      success("Planning complete!")
      console.log(`  Plan: ${finalTask.planPath}`)
      console.log(`  View: dash-ai tasks plan-docs ${id}`)
    }
  })
```

**Key behavior:**
- Always runs in embedded mode (planning needs Pi SDK)
- Uses the server's existing `POST /api/tasks/:id/start-planning` endpoint
- Polls for completion (the queue worker + planningRunner handle the actual session)
- Shows spinner with tool call progress in human mode
- Returns structured JSON in `--json` mode

### 2. `dash-ai tasks plan <id> --feedback "..."`

Iteration mode — calls `/api/tasks/:id/iterate-plan` with feedback text. The planningRunner handles sending `/skill:start-work-plan` with the feedback context.

### 3. `dash-ai tasks plan-docs <id>`

```bash
dash-ai tasks plan-docs <id>                       # list available plan docs
dash-ai tasks plan-docs <id> --file BRIEF          # print BRIEF.md content
dash-ai tasks plan-docs <id> --file ROADMAP        # print ROADMAP.md content
dash-ai tasks plan-docs <id> --file phases/01-.../01-01-PLAN.md
dash-ai tasks plan-docs <id> --stdout              # pipe all docs to stdout (agent mode)
```

Implementation:
- `GET /api/tasks/:id/plan-doc?file=<name>` (existing endpoint)
- Without `--file`: list available files discovered from `.planning/` directory
- With `--file`: print file content (markdown rendered or raw)
- `--stdout`: concatenate all plan docs with separators for agent consumption

### 6. `dash-ai tasks review <id>` — Reviewer Persona

New command. Runs a reviewer persona that analyzes the diff and plan, producing a structured summary:

```typescript
tasks.command("review <id>")
  .description("Run reviewer persona on completed coding session")
  .option("--persona <name>", "reviewer persona to use")
  .action(async (id, opts) => {
    const ctx = getContext(opts)
    const { client } = await resolveClient(ctx)

    const task = await client.get(`/api/tasks/${id}`)
    if (task.status !== "AWAITING_REVIEW") {
      error(`Can only review tasks in AWAITING_REVIEW status. Current: ${task.status}`, { exitCode: 1 })
    }

    // Get the diff
    const diff = await client.get(`/api/tasks/${id}/diff`)

    // Get the plan docs for context
    const brief = await client.get(`/api/tasks/${id}/plan-doc?file=BRIEF.md`)

    // Run reviewer via embedded Pi SDK session
    const review = await runReviewerSession({
      taskId: id,
      diff: diff.content,
      brief: brief.content,
      persona: opts.persona ? await resolvePersona(client, opts.persona) : await getDefaultReviewer(client),
    })

    if (ctx.json) {
      console.log(formatJson(review))
    } else {
      // Pretty-print review summary
      console.log(chalk.bold("Review Summary"))
      console.log(review.summary)
      console.log()
      if (review.concerns?.length) {
        console.log(chalk.yellow("Concerns:"))
        review.concerns.forEach(c => console.log(`  - ${c}`))
      }
    }
  })
```

**Reviewer persona runner** (`src/embedded/reviewerRunner.ts` — new):
- Creates a Pi SDK session with read-only tools (no code modification)
- System prompt instructs the model to compare the diff against the plan
- Returns structured JSON: `{ summary, filesChanged, linesAdded, linesRemoved, concerns, matchesPlan: boolean }`

### 8. Polling helper (`src/api/poll.ts`)

Shared utility for commands that need to wait for status changes:

```typescript
export async function pollTaskStatus(
  taskId: string,
  client: DashAiClient,
  options: {
    targetStatus: string[]
    spinner?: Ora
    onEvent?: (event: any) => void
    intervalMs?: number
    timeoutMs?: number
  }
): Promise<Task> {
  const interval = options.intervalMs ?? 2000
  const timeout = options.timeoutMs ?? 20 * 60 * 1000  // 20 min
  const start = Date.now()

  while (true) {
    const task = await client.get(`/api/tasks/${taskId}`)
    if (options.targetStatus.includes(task.status)) return task
    if (Date.now() - start > timeout) throw new Error("Polling timed out")
    await sleep(interval)
  }
}
```

### 9. Embedded runner wrapper (`src/embedded/runner.ts`)

Thin wrapper that sets up the embedded server for commands that need Pi SDK access:

```typescript
export async function withEmbeddedServer<T>(
  fn: (client: DashAiClient) => Promise<T>
): Promise<T> {
  const { url, token } = await ensureEmbeddedServer()
  const client = new DashAiClient(url, token)
  try {
    return await fn(client)
  } finally {
    await teardownEmbeddedServer()
  }
}
```
