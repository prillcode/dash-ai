<objective>
Create database table for AI model configurations to replace the models.json file.
This enables users to define and manage AI models (provider, model ID, API key optional)
via a UI, integrating with the existing persona system for model selection.

Purpose: Centralize model configuration in the database for easier management,
allow UI-based model setup, and pave the way for the Config menu feature.

Output: New `models` table schema, migration, service functions, and integration
with existing models.json fallback. `pnpm --filter server build` clean.
</objective>

  <action>
    Design a `models` table with fields:
    - `id` (text primary key, generated)
    - `name` (text, user-friendly name)
    - `provider` (text, e.g., "anthropic", "deepseek", "openai", "zai")
    - `modelId` (text, e.g., "claude-sonnet-4.6", "deepseek-reasoner", "glm-4.7")
    - `apiKey` (text optional, encrypted? For now store plaintext with warning)
    - `isActive` (boolean, default true)
    - `createdAt`, `updatedAt` (text timestamps)
    - `config` (text JSON, for extra provider-specific settings)

    Consider relationship with personas: personas reference model by provider+modelId
    (currently personas have separate model and provider fields). We'll keep that
    for now; later we can add foreign key.

    Update `packages/server/src/db/schema.ts` with the new table definition.
    Follow existing pattern (see `personas`, `projects` tables).

    **Additive migration rule**: Create a new migration file in `packages/server/src/db/migrations/`
    with SQL to create the table. Do NOT wipe the database. Use manual migration
    as per AGENTS.md guidelines.

    **Migration steps:**
    1. Create migration file `000X_create_models_table.sql` with CREATE TABLE.
    2. Update `_journal.json` and `0000_snapshot.json`.
    3. Run `pnpm db:migrate` to apply.

    **Note**: The existing `models.json` file at `~/.ai-dashboard/models.json` should
    be migrated: read its contents and insert into the new table (optional, can be done later).
