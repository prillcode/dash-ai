# Plan 02-01: Schema + Migration

## Goal
Replace the old 8-state TaskStatus enum with the new 9-state task machine and add
new fields: `planFeedback`, `planningPersonaId`, `planningPersonaName`, `codingPersonaId`
(rename from `personaId`), `planPath`. Run migration.

---

## Steps

### 1. Update `packages/server/src/db/schema.ts`

Replace the `TaskStatus` const and `tasks` table definition:

**New TaskStatus:**
```
DRAFT → IN_PLANNING → PLANNED → READY_TO_CODE → QUEUED → RUNNING → AWAITING_REVIEW → APPROVED / REJECTED → COMPLETE / FAILED
```

New const:
```typescript
export const TaskStatus = {
  DRAFT: "DRAFT",
  IN_PLANNING: "IN_PLANNING",
  PLANNED: "PLANNED",
  READY_TO_CODE: "READY_TO_CODE",
  QUEUED: "QUEUED",
  RUNNING: "RUNNING",
  AWAITING_REVIEW: "AWAITING_REVIEW",
  APPROVED: "APPROVED",
  REJECTED: "REJECTED",
  COMPLETE: "COMPLETE",
  FAILED: "FAILED",
} as const
```

**Tasks table field changes:**
- Change default status from `"PENDING"` to `"DRAFT"`
- Rename `personaId` → `codingPersonaId` (column: `coding_persona_id`)
- Rename `personaName` → `codingPersonaName` (column: `coding_persona_name`)
- Add `planningPersonaId` text nullable (column: `planning_persona_id`, references personas.id)
- Add `planningPersonaName` text nullable (column: `planning_persona_name`)
- Add `planFeedback` text nullable (column: `plan_feedback`)
- Add `planPath` text nullable (column: `plan_path`)

Keep all existing fields: `id`, `title`, `description`, `status`, `priority`,
`repoPath`, `targetFiles`, `sessionId`, `outputLog`, `diffPath`, `errorMessage`,
`reviewedBy`, `reviewNote`, `startedAt`, `completedAt`, `createdAt`, `updatedAt`.

Update indexes:
- Keep existing indexes but rename refs from `personaId` → `codingPersonaId`
- Add index on `planningPersonaId`

### 2. Run migration

```bash
export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh" && nvm use v22
cd packages/server
pnpm db:generate
pnpm db:migrate
```

Verify: migration SQL file created in `packages/server/src/db/migrations/`
and no errors from `pnpm db:migrate`.
