# Plan 02-02: Backend — taskService, queueWorker, routes

## Goal
Update backend services and routes to use the new 9-state task machine and new schema fields.

---

## Steps

### 1. Update `packages/server/src/services/taskService.ts`

**Update `TaskInput` interface:**
```typescript
export interface TaskInput {
  title: string
  description: string
  planningPersonaId?: string
  planningPersonaName?: string
  codingPersonaId: string
  codingPersonaName: string
  repoPath: string
  targetFiles?: string[]
  priority?: number
}
```

**Update `createTask`:**
- Initial status: `TaskStatus.DRAFT` (was `PENDING`)
- Map new fields: `codingPersonaId`, `codingPersonaName`, `planningPersonaId`, `planningPersonaName`
- No `personaId` / `personaName` anymore

**Update `parseTask`:**
- No changes to field names here (Drizzle returns column-mapped names)
- Ensure `targetFiles` JSON parse still works

**Update `claimNextPendingTask`:**
- Claim `READY_TO_CODE` tasks (not `PENDING`)
- Transition to `QUEUED`
- Rename function to `claimNextReadyTask` (keep old export as alias for now — update call sites in same PR)

**Update `resetStuckTasks`:**
- Also reset `IN_PLANNING` → `DRAFT` (in addition to `QUEUED`/`RUNNING` → `DRAFT`)
- Reset target status: `DRAFT` (was `PENDING`)

**Update `updateTaskStatus` signature:**
- Add `planFeedback`, `planPath`, `planningPersonaId`, `planningPersonaName` to the `extra` Partial pick

**Keep unchanged:** `listTasks`, `getTask`, `markTaskFailed`, `updateTaskDiffPath`

### 2. Update `packages/server/src/services/queueWorker.ts`

- Replace `claimNextPendingTask()` call with `claimNextReadyTask()`
- Update log message: `"Claimed task"` from `READY_TO_CODE` → `QUEUED`
- Update `appendEvent` calls: change `from: TaskStatus.PENDING` → `from: TaskStatus.READY_TO_CODE`
- Update `resetStuckTasks` log: mention `IN_PLANNING` reset too
- In `runTaskSession`, look up persona via `task.codingPersonaId` (not `task.personaId`)
- Pass `codingPersonaId` to `personaService.getPersona`

### 3. Update `packages/server/src/routes/tasks.ts`

**Update `taskSchema` (POST body):**
```typescript
const taskSchema = z.object({
  title: z.string().min(1),
  description: z.string().min(1),
  codingPersonaId: z.string().min(1),
  planningPersonaId: z.string().optional(),
  repoPath: z.string().min(1),
  targetFiles: z.array(z.string()).optional(),
  priority: z.number().min(1).max(5).optional(),
})
```

**Update `statusUpdateSchema`:**
- Replace `TaskStatus.PENDING` with `TaskStatus.DRAFT` in the allowed set
- Add `TaskStatus.PLANNED`, `TaskStatus.READY_TO_CODE` to the allowed set
- Full allowed set for manual status update:
  `DRAFT, PLANNED, READY_TO_CODE, APPROVED, REJECTED, FAILED`

**Update POST `/` handler:**
- Look up coding persona: `personaService.getPersona(parsed.data.codingPersonaId)`
- Optionally look up planning persona if `planningPersonaId` provided
- Pass `codingPersonaId`, `codingPersonaName`, `planningPersonaId`, `planningPersonaName` to `createTask`
