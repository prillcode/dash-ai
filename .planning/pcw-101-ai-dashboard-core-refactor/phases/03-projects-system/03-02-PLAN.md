    In `schema.ts`, add the `projects` table after `personas` and before `tasks`:

    ```typescript
    export const projects = sqliteTable("projects", {
      id: text("id").primaryKey(),
      name: text("name").notNull(),
      description: text("description").notNull().default(""),
      path: text("path").notNull(),
      isActive: integer("is_active", { mode: "boolean" }).notNull().default(true),
      createdAt: text("created_at").notNull(),
      updatedAt: text("updated_at").notNull(),
    }, (table) => ({
      isActiveIdx: index("idx_projects_is_active").on(table.isActive),
      createdAtIdx: index("idx_projects_created_at").on(table.createdAt),
    }))
    ```

    happens at validation time and runtime, not at storage time.

<task type="auto">
  <name>Task 2: Create `projectService.ts`</name>
  <files>
    packages/server/src/services/projectService.ts

  <action>
    Create `packages/server/src/services/projectService.ts` modelled after `personaService.ts`.

    **Types:**
    ```typescript
    export interface ParsedProject {
      id: string
      name: string
      description: string
      path: string          // as stored (may contain ~)
      resolvedPath: string  // ~ expanded via os.homedir()
      isActive: boolean
      createdAt: string
      updatedAt: string
    }

    export interface ProjectInput {
      name: string
      description?: string
      path: string
    }
    ```

    **Functions:**
    - `listProjects(activeOnly = true): Promise<ParsedProject[]>` — order by createdAt desc
    - `getProject(id: string): Promise<ParsedProject | null>`
    - `createProject(input: ProjectInput): Promise<ParsedProject>`
    - `updateProject(id: string, input: Partial<ProjectInput>): Promise<ParsedProject | null>`
    - `deleteProject(id: string): Promise<boolean>` — hard delete (unlike personas which soft-delete)
    - `validateProjectPath(path: string): { valid: boolean; resolvedPath: string; error?: string }`
      - Resolve `~` via `os.homedir()` (import from "os")
      - Use `existsSync` + `statSync` from "fs" to check dir exists and is a directory
      - Returns `{ valid: true, resolvedPath }` or `{ valid: false, resolvedPath, error: "..." }`

    **`parseProject` helper:** expand `~` in `path` to produce `resolvedPath` for every returned row.

    Use `generateId()` and `now()` from utils (same as other services).
  </action>

    files not yet updated — projectService itself must be error-free).
  </verify>
