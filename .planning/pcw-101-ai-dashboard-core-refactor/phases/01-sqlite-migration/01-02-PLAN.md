<task type="auto">
  <name>Task 1: Clear old migrations and generate fresh SQLite migration</name>
  <files>packages/server/src/db/migrations/</files>
  <action>
    The existing migration was generated for the Turso dialect and will not work with SQLite.
    Delete all existing migration files and metadata, then regenerate:

    ```bash
    rm -rf packages/server/src/db/migrations/
    mkdir -p packages/server/src/db/migrations/
    # Create the ~/.ai-dashboard directory if it doesn't exist
    mkdir -p ~/.ai-dashboard
    # Generate fresh migration for SQLite
    pnpm db:generate
    ```

    This will produce a new 0000_*.sql file compatible with better-sqlite3.
    Do NOT run db:migrate yet — do that after env is updated.
  </action>
  <verify>ls packages/server/src/db/migrations/ && cat packages/server/src/db/migrations/*.sql | head -20</verify>
  <done>New migration file exists, contains CREATE TABLE statements for personas, tasks, task_events</done>
</task>

<task type="auto">
  <name>Task 2: Update .env and .env.example</name>
  <files>.env, .env.example</files>
  <action>
    Update .env.example — replace Turso section with SQLite:

    ```bash
    # SQLite (local file, auto-created on first run)
    # Default: ~/.ai-dashboard/dashboard.db
    # Override with an absolute or ~/relative path:
    # SQLITE_DB_PATH=~/.ai-dashboard/dashboard.db

    # OpenCode working directory
    OPENCODE_WORKING_DIR=/home/user/agent-workspaces

    # Local storage for diffs and session logs
    DIFF_STORAGE_DIR=~/.ai-dashboard/diffs
    LOG_STORAGE_DIR=~/.ai-dashboard/sessions

    # Skills base directory (for file-based skills path validation)
    SKILLS_BASE_DIR=~/.ai-dashboard/skills

    # Auth — all API requests require: Authorization: Bearer <API_TOKEN>
    API_TOKEN=change-me

    # Queue worker
    MAX_CONCURRENT_SESSIONS=3

    # Server
    PORT=3000
    NODE_ENV=development

    # Client (Vite exposes VITE_ prefixed vars to browser)
    VITE_API_TOKEN=change-me
    ```

    Update .env — remove TURSO_DATABASE_URL, TURSO_AUTH_TOKEN, TURSO_LOCAL_DB_PATH lines.
    Leave all other existing values (API_TOKEN, VITE_API_TOKEN, PORT, etc.) intact.
    Do NOT add SQLITE_DB_PATH to .env — the default path (~/.ai-dashboard/dashboard.db) is fine.
  </action>
  <verify>grep -E "TURSO|libsql" .env .env.example || echo "clean - no Turso vars"</verify>
  <done>No TURSO_* vars in either file, .env.example has SQLite comment block</done>
</task>
