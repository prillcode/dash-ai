<objective>
Implement the coding-mode session runner: when a task is `READY_TO_CODE` (and
gets claimed to `QUEUED`/`RUNNING`), the runner invokes `/run-plan` on the
appropriate PLAN.md file in the project repo via the same OpenCode SDK/CLI approach
established in plan 04-01. Add robust auth-error detection that surfaces a
clear user message instead of a raw SDK exception.

Purpose: Completes the execution side of the workflow. After planning produces
PLAN.md files, coding must be able to pick them up and run them. Auth errors
are the most common silent failure — catching them explicitly is critical.

Output: `packages/server/src/opencode/codingRunner.ts` — mirrors `planningRunner.ts`
interface but invokes `/run-plan`; `sessionRunner.ts` refactored to delegate to
`codingRunner`; `queueWorker.ts` updated to use the new coding runner. Auth error
patterns detected + surfaced via `errorMessage`. `pnpm --filter server build` clean.
</objective>

    ```typescript
    export interface CodingRunnerInput {
      taskId: string
      taskTitle: string
      taskDescription: string
      repoPath: string          // resolved (no ~)
      planPath: string          // e.g. "pcw-101-my-task" — relative to repoPath/.planning/

        model: string
        systemPrompt: string
        allowedTools: string[]
      }

      success: boolean
      sessionId: string
      diffPath?: string
      logPath?: string
      errorMessage?: string
    }

    export async function runCodingSession(
      input: CodingRunnerInput,
      onEvent: (type: string, payload: Record<string, unknown>) => Promise<void>
    ): Promise<CodingResult>
    ```

    **Prompt construction:**
    The prompt sent to Claude must include:
    - "You are working in the repo at [repoPath]."
    - "Run the plan at .planning/[planPath]/ — find the first unexecuted PLAN.md
       file (no corresponding SUMMARY.md) and execute it using the run-plan skill."
    - System prompt from `codingPersona.systemPrompt`

    If a `run-plan` slash command exists at `~/.claude/commands/run-plan.md`,
    reference it. If not, instruct Claude to read the PLAN.md directly and execute
    the tasks defined within it.

    **Auth error detection:**
    Wrap the entire session in try/catch. Additionally, monitor streamed events for
    auth failure patterns. Auth failures manifest in several ways depending on SDK/CLI:
    - Error message containing: `"authentication"`, `"401"`, `"403"`, `"invalid_api_key"`,
      `"ANTHROPIC_API_KEY"`, `"unauthorized"`, `"permission denied"` (case-insensitive)
    - Exit code 401 or 403 (CLI mode)

    If any of these are detected, return:
    ```typescript
    {
      success: false,
      sessionId: "",
      errorMessage: "Authentication failed. OpenCode SDK reads credentials from ~/.local/share/opencode/auth.json.\n" +
                    "Use '/connect' in OpenCode TUI/IDE to add providers, or set environment variables (ANTHROPIC_API_KEY, DEEPSEEK_API_KEY, etc.)."
    }
    ```

    **Log + diff capture:**
    Capture all streamed output to a log file at `~/.ai-dashboard/sessions/[taskId]/session.log`.
    Attempt to read `git diff HEAD` from the repoPath after session completes and save to
    `~/.ai-dashboard/diffs/[taskId]/changes.diff`. If `git diff` fails, skip gracefully.

    Use `os.homedir()` to resolve `~` in all paths.
    Use `fs.promises.mkdir(..., { recursive: true })` before writing.

    **`sessionRunner.ts` refactor:**
    The existing `SessionRunner` class and `runSession` function are placeholders.
    Replace the body of `runSession` to delegate to `codingRunner`:
    ```typescript
    import { runCodingSession } from "./codingRunner"

    export async function runSession(
      task: { id: string; title: string; description: string; repoPath: string; planPath?: string | null },
      persona: { id: string; name: string; model: string; systemPrompt: string; allowedTools: string[]; contextFiles: string[] }
    ): Promise<SessionResult> {
      const result = await runCodingSession(
        {
          taskId: task.id,
          taskTitle: task.title,
          taskDescription: task.description,
          repoPath: task.repoPath,
          planPath: task.planPath || "",
          codingPersona: {
            id: persona.id,
            name: persona.name,
            model: persona.model,
            systemPrompt: persona.systemPrompt,
            allowedTools: persona.allowedTools,
          },
        },
        async (type, payload) => {
          await eventService.appendEvent(task.id, type, payload)
        }
      )

      if (!result.success) {
        throw new Error(result.errorMessage || "Coding session failed")
      }

      return {
        diffPath: result.diffPath || "",
        logPath: result.logPath || "",
        sessionId: result.sessionId,
      }
    }
    ```
    Keep the `SessionRunner` class stub for backward compat but have its `run()` delegate
    to `runSession()` as well.
