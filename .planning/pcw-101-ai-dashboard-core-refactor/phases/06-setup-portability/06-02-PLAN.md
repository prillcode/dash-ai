<task type="auto">
  <name>Task 1: Run full build + fix all TypeScript errors</name>
  <files>
    [Any files with TypeScript errors — determined at runtime]
  </files>
  <action>
    Run the full monorepo build:
    ```bash
    export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh" && nvm use v22
    pnpm build 2>&1
    ```

    Collect ALL TypeScript errors from the output. Fix them systematically:

    **Common error patterns and fixes:**

    1. **Missing type imports** — add the import
    2. **Property does not exist on type** — check if a new field was added to DB schema
       but not to the corresponding TypeScript interface (or vice versa)
    3. **Argument of type X is not assignable to Y** — add type assertion or fix the
       type definition
    4. **Cannot find module** — check import path, add missing file, or fix path alias
    5. **Object is possibly null/undefined** — add null check or non-null assertion
       (use `!` sparingly; prefer explicit null checks)
    6. **Type X has no properties in common with type Y** — likely a mismatched interface
       between client types and server response shape

    **Do NOT suppress errors with `// @ts-ignore` or `as any` unless:**
    - The type is genuinely unknowable at compile time (e.g., event payloads from SDK)
    - AND the runtime behavior is correct
    - AND a TODO comment explains what the real fix would be

    After fixing, re-run `pnpm build` until it exits 0.

    If there are errors that would require significant architecture changes to fix
    (Rule 4 — architectural deviation), STOP and ask the user before proceeding.

    For non-breaking improvements (nice-to-haves), log them to
    `.planning/pcw-101-ai-dashboard-core-refactor/ISSUES.md` and continue.
  </action>
  <verify>
    `pnpm build` exits 0.
    Zero TypeScript errors in output.
    No `// @ts-ignore` comments added without TODO explanation.
  </verify>
  <done>
    Full monorepo build passes with zero TypeScript errors.
    Any suppressed errors have explanatory TODO comments.
  </done>
</task>
