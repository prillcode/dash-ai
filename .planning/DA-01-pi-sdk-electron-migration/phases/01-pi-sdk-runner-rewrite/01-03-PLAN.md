# PLAN 01-03 — Rewrite codingRunner.ts with Pi SDK + start-work-run Skill

## Objective
Replace the `gnome-terminal` + `opencode-cli run` coding session with an in-process Pi SDK `AgentSession` that invokes `/skill:start-work-run` for structured, phased code execution. Coding sessions get full tool access (read, bash, edit, write) and stream real-time activity to the UI.

## Key Design Decisions

### Skill invocation
The coding runner no longer sends a flat task description prompt. Instead, it invokes `/skill:start-work-run`, which:
- Walks through all phase plans in `.planning/<id>-<name>/phases/` sequentially
- Each PLAN.md has verification steps and done conditions
- Checkpoints are produced as each phase completes
- The skill handles its own progress tracking internally

This gives Dash AI structured, verified execution instead of a single-shot prompt.

### Tools for coding
Coders need full source modification capabilities:
- Use `createCodingTools(repoPath)` — gives read, bash, edit, write
- These are scoped to `repoPath` automatically via factory functions

### System prompt
Use the persona's `systemPrompt` field. The skill content is appended by Pi's skill discovery.

### Diff capture
Keep the current `git diff HEAD` approach — run it after the Pi session completes.

## Tasks

### 1. Rewrite `runCodingSession()` in `codingRunner.ts`

```typescript
import {
  createAgentSession,
  SessionManager,
  createCodingTools,
  DefaultResourceLoader,
  SettingsManager,
} from "@mariozechner/pi-coding-agent"
import { resolveModel, checkProviderAuth } from "./piSession"

export async function runCodingSession(
  input: CodingRunnerInput,
  onEvent: (type: string, payload: Record<string, unknown>) => Promise<void>
): Promise<CodingResult> {
  const sessionDir = join(homedir(), ".dash-ai", "sessions", input.taskId)
  const diffDir = join(homedir(), ".dash-ai", "diffs", input.taskId)
  const diffPath = join(diffDir, "changes.diff")

  await mkdir(sessionDir, { recursive: true })
  await mkdir(diffDir, { recursive: true })

  try {
    await onEvent("CODING_EVENT", { status: "starting", message: "Starting coding session" })

    // Auth pre-flight
    const providerID = input.codingPersona.provider || "anthropic"
    const authCheck = await checkProviderAuth(providerID)
    if (!authCheck.ok) throw new Error(authCheck.errorMessage!)

    // Resolve model
    const model = await resolveModel(providerID, input.codingPersona.model)

    await onEvent("CODING_EVENT", {
      status: "launching",
      model: `${model.provider}/${model.id}`,
    })

    // Build system prompt from persona — Pi appends skill content automatically
    const loader = new DefaultResourceLoader({
      cwd: input.repoPath,
      systemPromptOverride: () => input.codingPersona.systemPrompt,
    })
    await loader.reload()

    const { session } = await createAgentSession({
      cwd: input.repoPath,
      model,
      thinkingLevel: "medium",
      tools: createCodingTools(input.repoPath),
      resourceLoader: loader,
      sessionManager: SessionManager.inMemory(),
      settingsManager: SettingsManager.inMemory({ compaction: { enabled: true } }),
    })

    // Subscribe and forward events
    session.subscribe((event) => {
      switch (event.type) {
        case "message_update":
          if (event.assistantMessageEvent.type === "text_delta") {
            onEvent("CODING_TEXT", { delta: event.assistantMessageEvent.delta })
          }
          break
        case "tool_execution_start":
          onEvent("TOOL_START", { toolName: event.toolName, args: event.args })
          break
        case "tool_execution_end":
          onEvent("TOOL_END", { toolName: event.toolName, isError: event.isError })
          break
        case "turn_start":
          onEvent("TURN_START", { turnIndex: event.turnIndex })
          break
        case "turn_end":
          onEvent("TURN_END", { turnIndex: event.turnIndex })
          break
        case "agent_end":
          onEvent("CODING_EVENT", { status: "completed" })
          break
      }
    })

    // Invoke start-work-run to execute all phase plans
    // The skill walks through phases sequentially with verification
    await session.prompt(
      [
        `/skill:start-work-run`,
        `Plan path: ${input.planPath}`,
        `Task: ${input.taskTitle}`,
        ``,
        input.taskDescription,
      ].join("\n")
    )

    // Capture git diff after session completes
    const diff = await captureGitDiff(input.repoPath, diffPath)

    session.dispose()

    return {
      success: true,
      sessionId: session.sessionId,
      diffPath: diff ? diffPath : undefined,
      logPath: join(sessionDir, "session.log"),
    }
  } catch (error) {
    // ... error handling, return CodingResult with errorMessage
  }
}
```

### 2. Extract `captureGitDiff()` helper
- Shared utility used after session completion
- Runs `git diff HEAD` in the repo
- Writes output to diffPath if non-empty
- Returns boolean indicating whether diff was captured
- Unchanged from current logic — just extracted for clarity

### 3. Remove all `gnome-terminal` and `opencode-cli` code
- Delete the `spawn("gnome-terminal", ...)` block
- Delete the temp shell script creation
- Delete the process exit polling / heartbeat interval
- Delete `normalizeModel()` import (now in `piSession.ts`)
- Remove the flat prompt construction (`"IMPORTANT: Do not ask..."`) — the skill provides structured execution instructions

### 4. Session timeout
- Add optional `setTimeout` wrapper that calls `session.abort()` after 20 minutes
- Simpler than current approach — no polling interval needed

### 5. Simplify or remove `sessionRunner.ts`
The `SessionRunner` class and `runSession()` function are thin wrappers around `runCodingSession()`. Remove the indirection — have `queueWorker.ts` call `codingRunner.ts` directly. This is addressed in 01-04.
