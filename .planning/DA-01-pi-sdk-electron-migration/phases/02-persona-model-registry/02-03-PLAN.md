## Objective
Update the server startup sequence to verify Pi SDK initialization, and update the frontend to use Pi's auth information for the auth warning banner and persona configuration.

## Tasks

### 1. Update server startup (`packages/server/src/index.ts`)

Replace the current `checkSkillsInstalled()` warning with a comprehensive startup check:

```typescript
import { existsSync } from "fs"
import { join } from "path"
import { homedir } from "os"
import { getAuth, getModelRegistry } from "./agent/piSession"

// 1. Check pi-native skills are installed
const requiredSkills = ["start-work-begin", "start-work-plan", "start-work-run"]
const missing = requiredSkills.filter(
  (s) => !existsSync(join(homedir(), ".agents", "skills", s))
)
if (missing.length > 0) {
  console.warn(`⚠️  Missing pi-native skills: ${missing.join(", ")}`)
  console.warn("    Planning and coding tasks will fail until skills are installed.")
  console.warn("    Expected location: ~/.agents/skills/")
}

// 2. Verify Pi SDK can initialize
try {
  const registry = getModelRegistry()
  const models = await registry.getAvailable()
  if (models.length === 0) {
    console.warn("⚠️  No AI models available. Configure API keys:")
    console.warn("    - Set ANTHROPIC_API_KEY, OPENAI_API_KEY, etc. env vars")
    console.warn("    - Or run: pi /login")
  } else {
    const providers = [...new Set(models.map(m => m.provider))]
    console.log(`Pi SDK initialized: ${models.length} models available from ${providers.join(", ")}`)
  }
} catch (err) {
  console.warn(`⚠️  Pi SDK initialization error: ${err.message}`)
}
```

### 2. Update `GET /api/auth/status` endpoint (new)

Add an endpoint that the frontend can call to check auth configuration status. This replaces the current `AuthWarningBanner` logic:

**`packages/server/src/routes/auth.ts`:**

```typescript
import { Hono } from "hono"
import { getModelRegistry, getAuth } from "../agent/piSession"

export const authRouter = new Hono()

authRouter.get("/status", async (c) => {
  const registry = getModelRegistry()
  const available = await registry.getAvailable()

  const providers = [...new Set(available.map(m => m.provider))]

  return c.json({
    configured: available.length > 0,
    availableProviders: providers,
    modelCount: available.length,
  })
})
```

This endpoint is already behind `authMiddleware` (for the Dash AI API token), so it's protected.
