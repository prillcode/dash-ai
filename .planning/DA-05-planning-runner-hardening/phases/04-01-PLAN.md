# Phase 04 — End-to-End Runner Verification and UX Follow-Through

## Objective
Verify the hardened planning and coding flows end-to-end using real task lifecycle transitions, and make any minimal UX follow-through changes needed so users can understand what the system selected and why.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- This phase should happen after the core server-side hardening work is in place.
- Recent runner fixes already improved correctness; this phase ensures the overall experience is stable and understandable.

## Tasks
1. Build an end-to-end verification matrix.
   - Cover fresh planning, failed planning, plan iteration, ready-for-code transition, coding execution, and artifact inspection.
2. Execute real-world verification scenarios.
   - Use at least one new task with a unique identifier.
   - Confirm planning target selection, plan artifact creation, coding target selection, and diff/session artifact quality.
3. Audit user-facing visibility.
   - Determine whether the current UI/API already shows enough session metadata.
   - If not, make minimal changes to expose the most important details (selected work item, selected execution doc, failure reason, runtime summary).
4. Check regression surfaces.
   - Ensure task status transitions still behave correctly.
   - Ensure event/history views remain coherent after telemetry changes.
5. Finalize rollout readiness.
   - Summarize what changed, what was verified, and any known follow-up items.

## Verification
- Fresh planning creates the intended work item and stores the correct `planPath`.
- Failed planning reports a clear failure and does not reuse another work item.
- Iteration updates the existing work item in place.
- Ready-for-code → coding selects the intended `PLAN.md`/`EXECUTION.md` target.
- Session/diff artifacts align with the selected plan.
- `pnpm build` passes.

## Done When
- The main task lifecycle is validated end-to-end.
- Any necessary small UX follow-through is complete.
- The hardening work is ready for regular use.
