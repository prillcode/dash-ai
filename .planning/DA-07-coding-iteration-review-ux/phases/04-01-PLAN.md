# Phase 04 — Diff Panel Replacement with Changed-Files Summary

## Objective
Replace or simplify the current raw unified diff preview so task detail shows a concise changed-files summary with added/deleted counts, which is more useful for agent review workflows while still allowing users to inspect diffs elsewhere if they prefer.

## Context
- See `../BRIEF.md`
- See `../ROADMAP.md`
- Primary files likely include `packages/client/src/components/diff/*`, relevant task APIs, and any server helpers needed to summarize diff artifacts.

## Tasks
1. Audit the current diff review panel.
   - Identify how the raw diff artifact is fetched and rendered today.
   - Decide whether summary data should be derived client-side from the diff or returned via a server helper.
2. Design the changed-files summary.
   - Show file path plus added/deleted line counts.
   - Keep the default review surface concise and readable.
3. Implement the summary UI.
   - Replace or demote the raw `diff --git` dump in task detail.
   - Preserve access to underlying diff artifacts only if still useful as an advanced/secondary view.
4. Validate usefulness.
   - Ensure the summary is enough for quick triage while letting users open the repo in their IDE for actual patch review.

## Verification
- Task detail no longer defaults to a raw unified diff dump.
- Changed files display includes added/deleted counts per file.
- The new summary remains understandable for multi-file runs.
- `pnpm build` passes.

## Done When
- Changed-file summary replaces the low-value raw diff panel as the default review experience.
- Users can quickly see what changed without wading through the full patch in Dash AI.
